## 常用操作

1. 在子分支开发完后，合并到主分支之前，在子分支中先更新到主分支最新
```shell
git pull --rebase origin <主分支名>
```
2. 假如所有功能开发都在某个子分支，最终会合入主干分支（下面全在子分支操作）
- 每次开发新功能前
```shell
git pull --rebase origin <子分支名>
```
- 开发完合入主分支前
```shell
git pull --rebase origin <主分支名>
```
